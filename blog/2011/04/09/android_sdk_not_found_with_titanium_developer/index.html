    Titanium DeveloperでAndroid SDKが見つからないときの対処メモ

    Titatium Developer でなぜかAndroid SDKが見つからないといって困ったので、対処方法のメモ。きちんと動作するかは知らない。

    環境はこんな感じ

    • OSX10.6.7
    • Titanium Developer1.2.2
    • Xcode4.0.1
    • AndroidはバージョンがよくわからないけどAndroid SDK Tools, revision 10とAndroid SDK Platform-tools, revision 3 とSDK Platform Android 3.0 API11, revision1ほか

    症状

    Titanium DeveloperでNew Project→Project type:Mobileを選択すると、Xcodeは自動認識するもAndroid SDKを認識せず、Android SDKをインストールしたディレクトリを指定しても見つからないと言われる。

    解決方法

    Android SDKを配置したパスが /Library/Frameworks/android-sdk-mac_x86 だとした場合、以下のようにadbのシンボリックリンクを貼る。

    $ ln -s /Library/Frameworks/android-sdk-mac_x86/platform-tools/adb /Library/Frameworks/android-sdk-mac_x86/tools/adb

    すると、Android SDKをインストールしたディレクトリとして /Library/Frameworks/android-sdk-mac_x86 を選択すれば認識はされる。
